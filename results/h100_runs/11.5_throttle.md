# 11.5 — Power/thermal throttle check (NEGATIVE — no throttle observed)

**Date:** 2026-05-03 19:59-20:05
**GPU:** 1× NVIDIA H100 NVL
**Workload:** SF100 sort in tight loop for 5 min, telemetry every 1 sec
**Status:** PASS (no throttle in 5 min)

## Measurements

```
Total runs:      4 (each took ~30-42 s wall, mostly cold-cache CPU work)
Sampling:        nvidia-smi -lms 1000 (1 s interval)
Total samples:   404

First 30 s avg:  clock=1054 MHz, temp=20.2°C, power=87.9 W
Last 30 s avg:   clock=1980 MHz, temp=22.0°C, power=117.1 W
Drift:           clock +926 MHz, temp +1.8°C, power +29.2 W

Peak temp:       28°C
Min clock:       172 MHz (during between-run idle)
Max clock:       2055 MHz (during CUB radix sort phase)

Wall time first 3 avg: 36349 ms
Wall time last 3 avg:  35586 ms (-2.1% drift = noise)
```

## Interpretation

**No throttle observed in 5 min sustained.** GPU temperature stayed
under 30°C (huge headroom — H100 NVL TjMax is 105°C). Average
power was only 88-117 W vs the 350 W TDP because each sort run is
mostly CPU work (compact extract + gather) with the GPU idle between
short bursts of HBM-saturated radix sort.

The "drift" of clock going *up* (1054 → 1980 MHz) reflects when the
sample window happened to align with the GPU's active phase, not
thermal behavior. The actual GPU peak was 2055 MHz (the H100 NVL's
boost clock). No throttle to base clock.

## Conclusion

The H100 NVL on this box has plenty of thermal + power headroom for
sustained sort workloads. **A 30-min or 60-min loop test would
likely show no degradation either.** The GPU isn't the bottleneck
of our wall time; it's the host-side gather. So thermal throttle
would only matter for compute-heavier workloads (e.g., training).

## Caveats

- The 5-min test had only 4 runs because each took 30-42 s of wall
  time (cold cache between runs). A more sustained test would
  preload the input or use a synthetic generator.
- Datacenter thermal management may kick in over hours; we didn't
  test sustained over 1 h.
- The 200 W headroom in observed avg power means the GPU is
  *not* the limit; pushing more concurrent kernels would not be
  thermal-bound.
