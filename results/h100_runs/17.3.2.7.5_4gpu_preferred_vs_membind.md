# 17.3.2.7.5 — 4-GPU partition: `--membind` BEATS `--preferred`

**Date:** 2026-05-04
**Workload:** SF500 (360 GB) split across 4 GPUs, 90 GB each
**Status:** PASS — confirms `--preferred` win is single-GPU-only

## Result

| Method | Best per-run aggregate wall | Aggregate GB/s |
|--------|------------------------------|------------------|
| 17.3.2.3 (--membind) | 5478 ms | **65.7** |
| 17.3.2.7.5 (--preferred) | 5851 ms | 61.5 |

**Slow-down: -6%.** `--preferred` is slightly worse for 4-GPU partition.

## Per-GPU detail (3 runs each)

| GPU | NUMA | run 1 | run 2 | run 3 |
|-----|------|-------|-------|-------|
| 0   | --preferred=0 | 6038 | 4947 | **3411** |
| 1   | --preferred=0 | **3845** | 5172 | 5421 |
| 2   | --preferred=1 | 5702 | 5878 | **5851** |
| 3   | --preferred=1 | 5520 | **5450** | 5539 |

Notice: GPU 2 and 3 never broke 5500 ms. With --membind in 17.3.2.3,
their best runs were 3879 (GPU 2) and 3665 (GPU 3). So GPU 2/3 are
~50% slower under --preferred than under --membind.

## Why the flip

In single-GPU SF300, the workload uses 470+ GB across one node;
node 1 has 500+ GB free; spilling there gets writes 2× DDR5
bandwidth from idle channels.

In 4-GPU SF500, **each node is already serving 2 GPUs at ~180 GB
input + 90 GB output ≈ 270 GB pinned + tmpfs**. There's no idle
DDR5 bandwidth on either node to spill to. Both nodes are already
saturated.

In that regime, `--preferred` adds runtime decision overhead
(checking page availability) for no benefit. `--membind` is
deterministic and wins.

## Updated deployment guidance (revised)

| Configuration | Best policy |
|---------------|-------------|
| 1 GPU, large workload (SF300+) | `--cpunodebind=0 --preferred=0` |
| 1 GPU, small workload (SF50/SF100) | `--cpunodebind=0 --preferred=0` (or --membind, same speed) |
| 2 GPU one-per-node | `--cpunodebind=N --preferred=N` (per GPU) |
| 4 GPU two-per-node | `--cpunodebind=N --membind=N` (per GPU pair) |

Rule: use `--preferred` when one node has free memory to spill to;
use `--membind` when both nodes are already saturated.

## Caveats

- Single sweep, 3 runs per GPU. Variance is significant (range
  2400 ms across runs for GPU 0).
- The flip between --preferred and --membind at 4-GPU is small
  (-6%) and could partially be noise. Want ≥3 sweeps to confirm.
