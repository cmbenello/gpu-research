# 18.4 — 2-GPU one-per-node concurrent SF50: zero contention

**Date:** 2026-05-04
**Setup:** GPU 0 → node 0, GPU 2 → node 1 (one per node)
**Workload:** Each GPU sorts SF50 (36 GB) independently
**Status:** PASS

## Result (--runs 3 each)

| GPU | NUMA | Cold | Warm 1 | Warm 2 |
|-----|------|------|--------|--------|
| 0 | node 0 | 1841 | 1435 | 1413 |
| 2 | node 1 | 1869 | 1416 | 1438 |

Last warm: 1413, 1438 ms.

## Aggregate throughput

Per-GPU at 25.0-25.5 GB/s = essentially identical to isolated 1-GPU
(25.5 GB/s from 18.2). **No contention** when each NUMA node has only
1 GPU active.

Aggregate: 2 × 36 GB / 1.438 sec = **50.1 GB/s**

## The contention scaling story (now complete)

| Config | per-GPU GB/s | aggregate | scaling efficiency vs 1-GPU |
|--------|---------------|-----------|------------------------------|
| 1 GPU SF50 | 25.5 | 25.5 | 1.00× (baseline) |
| **2 GPU one-per-node SF50** | **25.0-25.5** | **50.1** | **1.96× (98% efficient)** |
| 4 GPU 2-per-node SF50 (18.3) | 16.4 | 64.0 | 2.51× (63% efficient) |
| 4 GPU 2-per-node SF500 (17.3.2.3) | 16.4 | 65.7 | 2.58× (64% efficient) |
| 2 GPU one-per-node SF300 (17.3.2.3.2) | 25.4 | 50.8 | 2.00× (100%) |

## Conclusions

1. **Per-GPU sort throughput is determined by NUMA-node memory
   channel availability**, not by workload size. As long as one NUMA
   node serves only one active GPU, per-GPU stays at 25 GB/s.

2. **Pairing 2 GPUs on one NUMA node always costs 35%**. Same penalty
   at SF50 (16.4 GB/s/GPU) and SF500 (16.4 GB/s/GPU). So the
   contention is workload-independent — it's purely a memory
   bandwidth division.

3. **Best configurations:**
   - 1 GPU per box: 25.5 GB/s (single fast query)
   - 2 GPU one-per-node: 50.1 GB/s (no penalty)
   - 4 GPU 2-per-node: 64-66 GB/s (35% penalty, but more aggregate)

   Choice depends on what you're optimizing: latency (1-2 GPU),
   throughput (4-GPU), efficiency (2-GPU one-per-node).

## What this means for paper claims

Two clean throughput numbers to report:
- **1× H100 NVL: 32.7 GB/s on SF300** (single-GPU peak)
- **4× H100 NVL: 65.7 GB/s on SF500 4-way partition** (best aggregate)

Plus the in-between:
- **2× H100 NVL one-per-node: 50.8 GB/s on SF300/2** (no contention)

## Build & run

```bash
CUDA_VISIBLE_DEVICES=0 numactl --cpunodebind=0 --preferred=0 ./sort &
CUDA_VISIBLE_DEVICES=2 numactl --cpunodebind=1 --preferred=1 ./sort &
wait
```

## Caveats

- Only 3 runs per GPU. Variance unknown.
- The tiny per-GPU spread (1413 vs 1438) is within the 0.18% spread
  observed in 18.2 across all 4 GPUs.
- Both GPUs read the same input file; system page cache shared.
- Cold runs are slightly slower (1869 vs 1834 in 18.2 isolated)
  — possibly due to NUMA path setup competing.
