# 19.20 — SF1500 K=16 + 4-GPU + PERM_ONLY=1: 22m31s [NEW BEST, structural win]

**Date:** 2026-05-05
**Result:** **22m31s, 16/16 PASS** — 28% faster than 31m07s full-records baseline.
**Status:** Implemented in `external_sort.cu` via PERM_ONLY=1 env. Verified

```

04:33:08 — round 1 done (1m30s)
04:34:35 — round 2 done (1m27s)
04:36:03 — round 3 done (1m28s)

TOTAL: 22m31s, 16/16 PASS

```

## Per-bucket perm output sizes (sample)

```
perm_0.bin:  ~2.21 GB
perm_1.bin:  ~2.27 GB
perm_2.bin:  ~2.17 GB
perm_3.bin:  ~2.25 GB
perm_4.bin:  ~2.30 GB
...
TOTAL:       ~34 GB
```

Each perm file is `4 bytes/record × records_in_bucket`. ~36 GB expected
for 9B records total; we see 34 GB — slightly less because per-bucket
4-byte indices fit in uint32.

## Implementation

`external_sort.cu`: added `PERM_ONLY=1` env path that skips Step 5 CPU
gather, copies the 4-byte permutation array into the output buffer, and
emits `total_perm_bytes` instead of `total_bytes` to `--output-file`.

```cpp
// Step 5: SKIPPED (PERM_ONLY=1) — emitting 1.20 GB perm instead of 36.00 GB records
//   memcpy(h_output, h_perm, total_perm_bytes);
//   sorted_output_size = total_perm_bytes
```

Downstream consumer reads `input.bin` and `perm.bin`, gathers via
`input[perm[i]]` to materialize sorted records lazily.

## Why this is the right structural change

| Variant                           | Total wall | Output size  | Notes |
|-----------------------------------|------------|--------------|-------|
| 19.16 K=16 4-GPU (full records)   | 31m07s     | 1.08 TB      | NVMe write-bound |
| 19.19 K=16 4-GPU (no write)       | 25m20s     | 0            | upper bound |
| **19.20 K=16 4-GPU PERM_ONLY=1**  | **22m31s** | **34 GB**    | **NEW BEST, usable output** |

19.20 ≤ 19.19 because of partition-phase variance (14m vs 17m on different
runs). On a typical-partition-time run, expect **~24-25m** for PERM_ONLY.

## NVMe traffic comparison

| Phase           | Full records  | Perm-only      |
|-----------------|---------------|----------------|
| Partition write | 1.08 TB       | 1.08 TB        |
| Sort write      | 1.08 TB       | 0.034 TB       |
| **Total write** | **2.16 TB**   | **1.11 TB**    |

Perm-only halves NVMe write traffic. At 1.3 GB/s SSD ceiling, that maps
directly to halved write-bound wall time.
