# 18.5d — Sequential 1-GPU SF1000 sort: NEGATIVE (slower than concurrent)

**Date:** 2026-05-04
**Setup:** 4 buckets sorted sequentially on GPU 0, no parallelism
**Status:** PASS but NEGATIVE — slower than 18.5c 2-GPU concurrent

## Result

| Bucket | Wall (ms) | GB/s | Phase 1 | Phase 2 |
|--------|-----------|------|---------|---------|
| 0 | 26788 | 6.75 | 17341 | 9447 |
| 1 | 31514 | 5.66 | 17456 | 14058 |
| 2 | 32355 | 5.49 | 17362 | 14993 |
| 3 | 37618 | 4.87 | 17542 | 20076 |

Total: **682.76 sec = 11m23s** sort phase.

## Comparison

| Method | Sort phase wall | Notes |
|--------|------------------|-------|
| 18.5c (2-GPU one-per-node) | **374 sec = 6m14s** | best |
| 17.3.2.7.6 (4-GPU paired) | 713 sec = 11m53s | --preferred per pair |
| **18.5d sequential 1-GPU (this)** | **683 sec = 11m23s** | NEGATIVE |

Sequential gives up the parallelism without buying anything — cache
pollution still accumulates (Phase 2 wall grows from 9.4 sec at
bucket 0 to 20.1 sec at bucket 3, consistent with cache thrashing).

## What this rules out

- "Concurrent sorts hurt because of contention; sequential is faster":
  **FALSE.** Sequential is 1.83× slower than 2-GPU concurrent.
- "Cache pollution is the dominant effect": partly true (Phase 2
  growth from 9.4 → 20.1 sec confirms it), but the parallelism win
  outweighs cache pollution.

## What works (winning strategy reaffirmed)

**2-GPU one-per-node (18.5c)** at 6m14s remains the optimum:
- Concurrent sorting (2 GPUs in parallel)
- One per NUMA node (no memory contention)
- 2 rounds (avoids 4-GPU OOM)

## Build & run

```bash
bash /tmp/run_18.5d_sequential_sf1000.sh
```

## Caveats

- Single sweep.
- Phase 1 wall is NVMe-bound (~17 sec each, all very consistent).
- Phase 2 wall growth is the cache pollution signature.
- Adding `posix_fadvise(POSIX_FADV_DONTNEED)` between buckets MIGHT
  bring Phase 2 back to ~9 sec each, giving sequential ~104 sec
  total — competitive with 18.5c. Untested.

## Conclusion

For the paper: report 18.5c (6m14s sort phase) as the SF1000
pre-partitioned best. Sequential 18.5d is a useful ablation
showing that parallelism matters more than cache pollution
mitigation.
