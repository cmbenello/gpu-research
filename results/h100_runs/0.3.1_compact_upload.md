# 0.3.1 — slow-path compact-key upload [WIN, 1.5-1.7×]

**Date:** 2026-05-03 07:35 UTC
**GPU(s):** 1× NVIDIA H100 NVL (CUDA_VISIBLE_DEVICES=0)
**Commit:** `20f8058` "external_sort: 0.3.1 compact-key upload in slow-path single-pass"
**Status:** PASS

## Background

The slow-path single-pass code (data-fits-in-HBM but doesn't trigger
OVC) was uploading 66 B per record via strided cudaMemcpy2DAsync. PCIe
H2D was ~20 GB at SF50 / 40 GB at SF100, even though the GPU sort only
needed the 32 B compact key. Filed in 0.3 / BOTTLENECKS.md as the #2
bottleneck.

## What changed

Slow-path now does:
1. CPU-extract compact keys (32 B/record, only the bytes in
   `runtime_cmap`) into pinned host staging buffers.
2. **Pipelined**: 2 ping-pong staging buffers, 32 M records per chunk.
   While GPU H2D's chunk N, CPU extracts chunk N+1.
3. GPU sorts compact keys with `extract_uint64_from_compact_kernel`,
   doing 4 LSD passes (32 B / 8) instead of 9 (66 B / 8).
4. Stage buffers persist across `--runs` iterations as new sorter
   members `h_stage_persistent[2]` (avoids cudaMallocHost +
   cudaFreeHost overhead each iteration).

## Raw numbers (warm best of 5 runs)

| Scale  | Pre-0.3.1     | Post-0.3.1     | Speedup | PCIe before | PCIe after |
|--------|---------------|----------------|---------|-------------|------------|
| SF50   | 2.62 s @ 13.7 GB/s | **1.51 s @ 23.9 GB/s** | **1.74×** | 21.0 GB | 10.8 GB |
| SF100  | 4.50 s @ 16.0 GB/s | **3.02 s @ 23.8 GB/s** | **1.49×** | 42.0 GB | 21.6 GB |
| SF300  | 8.65 s @ 25.0 GB/s | (OVC path, untouched) | — | 64.8 GB | 64.8 GB |

PCIe compression: **1.94×** at SF50/SF100 (66 B → 32 B + LSD pass count
falls from 9 to 4).

## What worked

- Pipelining is the key. Without it, the CPU extract was serial and
  cost more than the PCIe savings. With ping-pong staging the CPU and
  PCIe overlap and the extract becomes essentially free.
- Stage-buffer persistence (mirroring B1's output-buffer reuse): a
  cudaMallocHost(1 GB) + cudaFreeHost cycle each `--runs` iteration
  was costing ~1 sec per iter. Persisting drops it to one-time.
- Headline: **SF100 (4.50 s → 3.02 s) now beats the RTX 6000 baseline
  of 3.74 s for the first time.** The throughput envelope chart should
  now be monotonic instead of dipping at SF50/100.

## What failed (false starts)

- **v0 (unpipelined CPU extract)**: just one CPU extract pass into one
  pinned buffer, then a single H2D. CPU extract took 4 s for 300 M
  records, completely wiping out the ~0.4 s PCIe savings. Net 2× regression.
- **v1 (pipelined, no persistence)**: pipelined extract+H2D got the
  per-chunk overlap working, dropping extract to ~500 ms after warmup.
  But cudaMallocHost(2 × 1 GB) + cudaFreeHost each run cost ~1 sec.
  Net win was tiny.
- **v2 (current)**: pipelined + persistent stage buffers across `--runs`.
  Clean 1.5-1.7× win.

## Surprises

1. **The compact CPU extract becomes 70-90 GB/s once warm** (raw record
   traffic / extract_ms reported by binary). That's most of the host
   memory bandwidth — i.e., we're memory-bound, not CPU-bound.
2. **Run-1 cold extract is ~17 s** for SF50 — much slower than warm
   (~500 ms). Cause: 36 GB of input reads from page-cold pages plus
   first-time pinned-buffer allocation (2 × 1 GB cudaMallocHost
   takes ~2 s on a fragmented host).
3. **The OVC path at SF300 doesn't go through this code** so no SF300
   change. The slight SF300 timing variance in this session vs prior
   measurements is page-cache state, not a code regression.

## Interpretation

This is the highest-leverage code fix yet on the single-GPU side. SF50
and SF100 both jump 1.5-1.7× without any algorithmic change —
literally just sending less data over PCIe. The remaining SF50/100
floor is now the GPU sort + CPU gather rather than upload. PCIe is
no longer the bottleneck below SF300.

Combined with the existing OVC path's compact upload at SF300+, the
ENTIRE single-GPU sort pipeline now uses 32 B keys end-to-end
regardless of scale. The throughput envelope chart should be
monotonic from SF10 (20 GB/s) → SF50 (24 GB/s) → SF100 (24 GB/s) →
SF300 (24-25 GB/s).
